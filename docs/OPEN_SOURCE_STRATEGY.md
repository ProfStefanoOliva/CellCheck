# Open Source Strategy

CellCheck is intended to follow a controlled open source strategy: open code, controlled brand.

## Core idea

- The software source code is intended to be open.
- The product identity remains controlled to avoid confusion about official and unofficial builds.
- The project has a didactic purpose and is designed for guided assessment workflows on spreadsheet exercises.

## Code versus product identity

The software license governs the source code. It does not automatically govern the right to present a modified build as the official CellCheck product.

That distinction is important because the project may allow forks while still protecting users from confusion about provenance, endorsement, maintenance, and quality expectations.

## Future direction

- A future Community Edition may be published under open source terms.
- Training, services, support, or institutional deployments may remain separate from the code license itself.
- Forks are acceptable, but they should avoid confusing users with the official project identity.

## Practical consequence

If someone redistributes a modified version, they should normally:

- use the open source code terms that apply to the project;
- choose a different product name for the modified distribution;
- avoid official CellCheck branding assets unless authorized;
- clearly state when a project is unofficial or based on CellCheck.

## Governance note

For final legal, trademark, or commercial structuring decisions, a qualified legal professional should be consulted.
