# Security Policy

## Reporting vulnerabilities

Please report suspected security issues through the official repository channels unless a dedicated security contact is later published.

Avoid opening public issues when the report may include sensitive details, private documents, or personal data.

## Sensitive file handling

- Do not attach real student workbooks.
- Do not attach files containing personal data.
- Do not attach institutional documents that should remain private.
- Prefer synthetic or anonymized examples whenever possible.

## Spreadsheet-specific note

CellCheck treats `.xlsm` workbooks in prudent read-only mode and does not execute macros.

Even so, vulnerability reports involving spreadsheet parsing should avoid sharing sensitive real-world files in public.
